# 政法领域企业智能知识库与AI Agent工作流平台

缩写简称：LAK-Agent（全局统一系统名、服务名、包名、域名）
完整英文全称：Legal Affairs Knowledge Agent Platform
中文官方名称：政法智能知识Agent平台

# 项目定位：
面向政法行业私有化部署智能问答平台，基于主Agent+多子Agent架构，完成用户诉求智能路由、政策RAG问答、办事指引RAG问答、投诉工单自动创建、低置信度人工兜底，满足政务等保要求、对话全链路审计、无幻觉合规答复。

# 技术栈：
- 基础环境：JDK17、Maven 3.8+
- 核心框架：Spring Boot 3.4.2
- AI框架：LangChain4j 1.14.0
- 向量数据库：Qdrant（独立服务，本地私有化部署）
- 大模型&向量模型：Qwen3.7-Max（百炼平台的对话大模型）、text-embedding-v4（百炼平台的向量化模型）
- 中间件：Redis 7（会话记忆+限流）、MySQL8.0（业务审计+工单数据）
- 工具依赖：Mybatis-Plus、Resilience4j熔断降级、SpringDoc接口文档


# 核心业务流程：
统一前置拦截 -> 主Agent意图识别+置信度判断 -> 路由分发至对应子Agent -> 业务处理 -> 结果合规校验 -> 统一结果封装返回 + 全链路日志落库
1. 政策咨询类：子AgentA → 政法政策库RAG检索 → 带原文溯源答复
2. 办事指引类：子AgentB → 办事指南库RAG检索 → 带原文溯源答复
3. 投诉建议类：子AgentC → 对话补齐工单必填项 → 调用内部工单接口创建工单 → 返回工单编号
4. 置信度<0.6 / 意图无法识别：统一兜底，返回人工客服联系方式，禁止AI自动作答

# 全局编码规范：
- 所有大模型入参、出参、检索片段全部落审计日志，满足政务6个月日志留存要求
- 所有AI答复必须携带文档来源、文件编号、生效时间，禁止无依据输出
- 全局添加政务敏感词双向校验（提问前置拦截、回答后置校验）
- 所有模型调用、第三方接口调用增加熔断、超时控制，防止服务雪崩
- 会话记忆持久化至Redis，不使用本地内存会话
