## 2. 快速开始（开发环境）

### 2.1 克隆项目

```bash
git clone <repository-url> LAK-Agent
cd LAK-Agent
```

### 2.2 启动基础设施

```bash
cd docker
docker compose up -d
```

等待所有容器健康检查通过：

```bash
docker compose ps
# 确认所有服务 STATUS 为 healthy
```

### 2.3 配置环境变量

```bash
# Linux / macOS
export DASHSCOPE_API_KEY="your-dashscope-api-key"
export JWT_SECRET="your-32-byte-jwt-secret-key-here"
export REDIS_HOST="localhost"
export MYSQL_URL="jdbc:mysql://localhost:3306/lak_ai_platform"

# Windows PowerShell
$env:DASHSCOPE_API_KEY="your-dashscope-api-key"
$env:JWT_SECRET="your-32-byte-jwt-secret-key-here"
$env:REDIS_HOST="localhost"
$env:MYSQL_URL="jdbc:mysql://localhost:3306/lak_ai_platform"
```

### 2.4 启动后端

```bash
cd backend
mvn clean package -DskipTests
java -jar target/lak-ai-platform-0.1.0-SNAPSHOT.jar
```

验证：

```bash
curl http://localhost:8080/api/v1/health
# {"status":"UP","service":"lak-ai-platform"}

curl http://localhost:8081/actuator/health
# {"status":"UP"}
```

### 2.5 启动前端

```bash
cd frontend
npm install
npm run dev
```

访问 `http://localhost:5173`，使用 `admin / admin123` 登录。

### 2.6 停止服务

```bash
# 停止前端:  Ctrl+C
# 停止后端:  Ctrl+C
# 停止基础设施:
cd docker
docker compose down          # 保留数据卷
docker compose down -v       # 删除所有数据（重置环境）
```

## 3. Docker 基础设施

### 3.1 docker-compose.yml 服务概览

| 服务 | 容器名 | 端口 | 数据卷 | 健康检查 |
|------|--------|------|--------|---------|
| MySQL | `lak-mysql` | 3306 | `mysql-data:/var/lib/mysql` | `mysqladmin ping` |
| Redis | `lak-redis` | 6379 | `redis-data:/data` | `redis-cli ping` |
| Qdrant | `lak-qdrant` | 6333, 6334 | `qdrant-data:/qdrant/storage` | `curl /health` |
| MinIO | `lak-minio` | 9000, 9001 | `minio-data:/data` | `curl /minio/health/live` |

### 3.2 环境变量 (.env)

```ini
# MySQL
MYSQL_ROOT_PASSWORD=root_dev_2026
MYSQL_DATABASE=lak_ai_platform
MYSQL_USER=lak_user
MYSQL_PASSWORD=lak_dev_2026
MYSQL_PORT=3306

# Redis
REDIS_PORT=6379

# Qdrant
QDRANT_REST_PORT=6333
QDRANT_GRPC_PORT=6334

# MinIO
MINIO_API_PORT=9000
MINIO_CONSOLE_PORT=9001
MINIO_ROOT_USER=lak_admin
MINIO_ROOT_PASSWORD=lak_dev_2026
```

> ⚠️ 生产环境务必修改所有密码。

### 3.3 MySQL 配置

```yaml
# docker-compose.yml 中的 command 参数:
--character-set-server=utf8mb4
--collation-server=utf8mb4_unicode_ci
--default-authentication-plugin=mysql_native_password
```

最大连接数使用默认值（151），HikariCP 连接池配置上限为 10。

### 3.4 Redis 配置

```yaml
# redis 启动参数:
--appendonly yes               # AOF 持久化
--maxmemory 256mb              # 最大内存
--maxmemory-policy volatile-lru  # 淘汰策略: 仅淘汰带 TTL 的 key
```

### 3.5 初始化脚本

将 SQL 脚本放置在 `docker/init-scripts/` 目录，MySQL 容器首次启动时自动执行。

## 5. 前端部署

### 5.1 构建

```

产物：`frontend/dist/` （纯静态文件）

# → http://localhost:5173
# Vite 代理: /api → http://localhost:8080
```

### 5.3 生产环境部署

将 `dist/` 部署到 Nginx：

```nginx
server {
    listen 80;
    server_name lak-agent.internal;

    # 前端静态资源
    root /opt/lak-ai-platform/frontend/dist;
    index index.html;

    # SPA 路由回退
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Trace-Id $request_id;

        # SSE 流式输出需要禁用缓冲
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 120s;
    }

    # Actuator 不对外暴露
    location /actuator {
        deny all;
    }
}
```

## 7. 数据库迁移 (Flyway)

### 7.1 迁移脚本

| 版本 | 文件 | 内容 |
|------|------|------|
| V1 | `V1__init_schema.sql` | 创建核心表：sys_user, sys_role, sys_permission, chat_session, chat_message, ticket, knowledge_document, audit_log_202606 |
| V2 | `V2__init_data.sql` | 种子数据：角色(ADMIN/OPERATOR/USER)、管理员账号(admin/admin123)、初始权限 |
| V3 | `V3__role_menu.sql` | 新增菜单体系：sys_menu, sys_menu_permission, sys_role_menu |
| V4 | `V4__audit_permission.sql` | 补充审计菜单权限 |

### 7.2 迁移行为

- Flyway 在应用启动时**自动执行**未应用的迁移脚本
- `baseline-on-migrate: true` — 已有数据库首次接入时自动建立基线
- 迁移脚本在 `classpath:db/migration/`，按版本号顺序执行
- **禁止修改已执行的迁移脚本** — 修改应通过新版本迁移完成

### 7.3 手动迁移（应急）

```bash
# 查看迁移状态
mvn flyway:info -Dflyway.url=jdbc:mysql://localhost:3306/lak_ai_platform \
  -Dflyway.user=root -Dflyway.password=root

# 手动执行迁移
mvn flyway:migrate

# 修复校验失败（谨慎使用）
mvn flyway:repair
```

## 9. 日志管理

### 9.1 日志级别

| 环境 | 框架日志 | 业务日志 | 安全日志 |
|------|---------|---------|---------|
| 开发 | DEBUG | DEBUG | INFO |
| 生产 | INFO | INFO | INFO |

### 9.2 日志格式

```
%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level [%X{traceId}] %logger{36} - %msg%n
```

每个请求自动携带 TraceId（通过 MDC 注入），可通过 `grep <traceId>` 回溯完整调用链。

### 9.3 审计日志管理

| 属性 | 说明 |
|------|------|
| 存储位置 | MySQL `audit_log_yyyyMM` 分表 |
| 在线保留 | 6 个月 |
| 归档周期 | 每月 1 日凌晨 02:00 |
| 归档格式 | CSV (UTF-8) |
| 归档路径 | `/data/archive/audit_log_yyyyMM.csv` |
| 法定保留 | 至少 2 年（归档文件形式） |

### 9.4 日志安全

- **禁止**在日志中输出 API Key、密码、Token
- **禁止**在日志中输出完整敏感词命中内容（仅记录"命中敏感词 + sessionId"）
- 生产环境禁止 `System.out.println()` 和 `e.printStackTrace()`（Hook 已配置拦截）

## 10. 备份与恢复

### 10.1 备份范围与频率

| 数据 | 备份方式 | 频率 | 保留 |
|------|---------|------|------|
| MySQL 业务数据 | `mysqldump` 全量 | 每日 | 30 天 |
| MySQL 审计日志 | `mysqldump` 按月分表 | 每月 | 6 个月在线 + 2 年归档 |
| Redis AOF | 文件拷贝 | 每日 | 7 天 |
| Qdrant 向量数据 | Snapshot API | 每周 | 4 周 |
| MinIO 文件 | `mc mirror` | 每日 | 30 天 |
| 敏感词配置文件 | Git 版本控制 | 变更时 | 永久 |

### 10.2 MySQL 备份

```bash
# 全量备份
mysqldump -h localhost -u root -p \
  --single-transaction \
  --routines \
  --triggers \
  --databases lak_ai_platform \
  > /data/backup/mysql/lak_ai_platform_$(date +%Y%m%d).sql

# 恢复
mysql -h localhost -u root -p lak_ai_platform \
  < /data/backup/mysql/lak_ai_platform_20260621.sql
```

### 10.3 Redis 备份

```bash
# Redis AOF 文件位于 /data/redis/appendonly.aof
# 备份只需拷贝该文件
cp /data/redis/appendonly.aof /data/backup/redis/appendonly_$(date +%Y%m%d).aof

# 恢复: 停止 Redis → 替换 AOF 文件 → 启动 Redis
```

### 10.4 Qdrant 备份

```bash
# 创建 Snapshot
curl -X POST http://localhost:6333/collections/lak_policy_docs/snapshots
curl -X POST http://localhost:6333/collections/lak_procedure_docs/snapshots

# 下载 Snapshot
curl http://localhost:6333/collections/lak_policy_docs/snapshots/<snapshot-name> \
  -o /data/backup/qdrant/lak_policy_docs_$(date +%Y%m%d).snapshot
```

### 10.5 自动化备份脚本（参考）

```bash
#!/bin/bash
BACKUP_DIR="/data/backup"
DATE=$(date +%Y%m%d)

# MySQL
mysqldump -h localhost -u lak_app -p"$MYSQL_PASSWORD" \
  --single-transaction lak_ai_platform \
  | gzip > "$BACKUP_DIR/mysql/lak_ai_platform_$DATE.sql.gz"

# 清理 30 天前的备份
find "$BACKUP_DIR/mysql" -name "*.sql.gz" -mtime +30 -delete
```

## 11. 运维操作手册

### 11.1 敏感词热加载

```bash
# 方式1: API 调用
curl -X POST http://localhost:8080/api/v1/admin/sensitive-words/reload \
  -H "Authorization: Bearer <admin-access-token>"

# 方式2: 直接修改文件后调用
vim config/sensitive-words.txt
# 然后调用 API 热加载
```

无需重启服务。加载过程中旧词库继续工作（`volatile` 无锁切换）。

### 11.2 知识库重新索引

```bash
# 单文档重新索引
curl -X POST http://localhost:8080/api/v1/knowledge/documents/DOC-20260621-0001/reindex \
  -H "Authorization: Bearer <admin-access-token>"
```

### 11.3 查看审计日志

```bash
# 分页查询（当前月）
curl "http://localhost:8080/api/v1/admin/audit-logs?page=1&size=20" \
  -H "Authorization: Bearer <admin-access-token>"

# 按状态筛选
curl "http://localhost:8080/api/v1/admin/audit-logs?status=FAIL" \
  -H "Authorization: Bearer <admin-access-token>"

# 查询指定月份
curl "http://localhost:8080/api/v1/admin/audit-logs?month=202605" \
  -H "Authorization: Bearer <admin-access-token>"
```

### 11.4 创建审计分表（手动）

若定时任务未配置，可手动创建下月分表：

```sql
CREATE TABLE IF NOT EXISTS `audit_log_202607` LIKE `audit_log_202606`;
```

### 11.5 查看 Qdrant Collection 状态

```bash
# 查看 Collection 列表
curl http://localhost:6333/collections

# 查看 Collection 详情
curl http://localhost:6333/collections/lak_policy_docs

# 统计 Point 数量
curl -X POST http://localhost:6333/collections/lak_policy_docs/points/count \
  -H "Content-Type: application/json" \
  -d '{}'
```

### 11.6 查看 Redis 会话

```bash
# 连接 Redis
redis-cli

# 查看所有会话 Key
KEYS session:*

# 查看特定会话
HGETALL session:<sessionId>

# 查看会话 TTL
TTL session:<sessionId>

# 手动关闭会话
HSET session:<sessionId> status CLOSED
```

### 11.7 优雅停机

```bash
# Spring Boot Actuator 优雅停机
curl -X POST http://localhost:8081/actuator/shutdown

# 或使用 kill（Spring Boot 默认启用优雅停机）
kill -15 <pid>
```

## 12. 故障排查

### 12.1 后端无法启动

| 症状 | 可能原因 | 排查方法 |
|------|---------|---------|
| `Connection refused: localhost:3306` | MySQL 未启动 | `docker compose ps` 检查 MySQL 状态 |
| `Connection refused: localhost:6379` | Redis 未启动 | `redis-cli ping` |
| `Connection refused: localhost:6334` | Qdrant 未启动 | `curl localhost:6333/health` |
| `DASHSCOPE_API_KEY not set` | 环境变量未配置 | `echo $DASHSCOPE_API_KEY` |
| Flyway 校验失败 | 迁移脚本被修改 | `mvn flyway:repair`（谨慎） |
| 端口 8080 被占用 | 其他进程占用 | `netstat -ano \| findstr 8080` (Win) / `lsof -i :8080` (Linux) |

### 12.2 对话无响应

| 症状 | 可能原因 | 排查方法 |
|------|---------|---------|
| 大模型超时 | 百炼 API 不可达 / 网络问题 | 检查防火墙出站规则 / `curl https://dashscope.aliyuncs.com` |
| RAG 检索返回空 | Qdrant 无数据 / 相似度阈值过高 | 检查 Qdrant Collection Point 数量 |
| 会话状态异常 | Redis 中的会话 TTL 过期 | `HGETALL session:<id>` 查看状态 |

### 12.3 审计日志查询报错

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| 查询其他月份返回空 | 该月分表未创建 | 手动 `CREATE TABLE ... LIKE` |
| 查询报 500 错误 | 月份参数格式错误 | 确保格式为 `yyyyMM` (6位数字) |
| 详情查询返回 404 | ID 不在当前月份表中 | 传入正确的 `?month=` 参数 |
